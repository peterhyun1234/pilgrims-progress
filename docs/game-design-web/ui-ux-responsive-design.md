## 1. 설계 원칙

### 1.1 핵심 원칙

1. **몰입 우선** — UI는 가능한 한 게임 세계의 일부처럼 느껴야 한다. 현대적 "유리판" 느낌 X, 양피지/고서적 질감 O
2. **미니멀 HUD** — 화면의 90% 이상은 게임 월드. HUD는 필요한 순간에만 강조
3. **터치 퍼스트, 키보드 동등** — 모바일에서 먼저 편해야 하고, 키보드/게임패드도 완전 지원
4. **44px 룰** — 모든 터치 타깃은 물리적 44px(11mm) 이상
5. **컨텍스트 적응** — 같은 UI도 상황(탐험/대화/전투/메뉴)에 따라 레이아웃과 크기가 변화
6. **접근성 내장** — 색약 모드, 폰트 크기 조절, 키보드 내비게이션은 옵션이 아니라 기본

### 1.2 참조 게임

| 게임 | 참조 요소 |
|------|----------|

## 3. 반응형 레이아웃

### 3.1 해상도 전략

게임의 내부 해상도는 **320x180** 고정이며, Phaser의 `Scale.FIT` 모드로 정수 배율 스케일링한다.

```
실제 렌더링 크기 = 320×180 × scale
scale = min(floor(viewportWidth / 320), floor(viewportHeight / 180))
최소 scale = 1 (320×180), 최대 scale = 제한 없음
```

UI 요소는 모두 내부 해상도(320x180) 기준으로 설계하되, **입력 영역(터치 타깃)은 물리 해상도 기준 44px 이상**을 보장한다.

### 3.2 디바이스 브레이크포인트

| 브레이크포인트 | 물리 해상도 범위 | scale 범위 | UI 모드 |
|--------------|-----------------|-----------|---------|
| **Mobile Portrait** | ~480×854 | 1–2 | `mobile-portrait` |
| **Mobile Landscape** | ~854×480 | 2 | `mobile-landscape` |
| **Tablet** | ~1024×768 | 3–4 | `tablet` |
| **Desktop** | 1280×720+ | 4–7 | `desktop` |

### 3.4 세이프 에리어 & 노치 대응

```css
/* CSS env() 사용 */
padding-top: max(env(safe-area-inset-top), 0px);
padding-bottom: max(env(safe-area-inset-bottom), 0px);
padding-left: max(env(safe-area-inset-left), 0px);
padding-right: max(env(safe-area-inset-right), 0px);
```

Phaser 게임 캔버스는 세이프 에리어 내부에 배치하고, HUD 및 터치 컨트롤은 세이프 에리어 여백을 반영하여 배치한다.

## 4. HUD 설계

### 4.1 스탯 바 상세

#### 구조 (각 스탯 바)

```
┌────────────────────────┐
│ [icon 8x8] ████████░░ │  ← 채워진 부분 + 빈 부분
│            FTH 65     │  ← 약어 + 수치 (선택적 표시)
└────────────────────────┘
```

- **크기**: 너비 48px, 높이 6px (내부 해상도 기준)
- **배치**: 화면 좌상단, 아이콘은 바 왼쪽
- **애니메이션**: 변화 시 목표값까지 0.5초에 걸쳐 부드러운 lerp
- **펄스**: 변화 직후 1초간 바 전체가 밝아졌다 원래로 (긍정: 금빛, 부정: 붉은빛)
- **수치 표시**: 데스크탑은 항상 표시, 모바일은 터치 시 3초간 표시

#### 짐 게이지 (Burden)

```
┌──────────────────────┐
│ [보따리] ▓▓▓▓▓▓▓▓░░ │  ← 높을수록 어두운 색
│          BRD 80      │
└──────────────────────┘
```

- **특수 연출**: burden ≥ 80이면 바가 미세하게 떨림 (1px 진동, 0.5Hz)
- **해방 연출**: burden → 0 시 바가 금빛으로 변한 뒤 1초 후 사라지고, "짐에서 해방!" 토스트

### 4.2 HUD 컨텍스트 모드

| 게임 상태 | HUD 표시 |
|----------|---------|
| 탐험 | 스탯 바 4개 (좌상단) |
| 대화 | 스탯 바 축소 → 아이콘만 (변화 시에만 확장) |
| 전투 | 스탯 바 → 전투 HUD로 전환 (HP 바, QTE 게이지) |
| 컷씬 | HUD 완전 숨김 (fadeOut 0.3초) |
| 메뉴/일시정지 | HUD 유지 + 오버레이 디밍 |

### 4.3 상호작용 프롬프트

```
       ╭───╮
  NPC  │ ! │  ← NPC 머리 위 (항상 표시)
       ╰───╯
       
       ╭─────────╮
       │ [E] 대화 │  ← 플레이어 근접 시 (상호작용 거리 24px 이내)
       ╰─────────╯
```

- **데스크탑**: `[E] 대화` / `[E] 조사` (키 + 동작명)
- **모바일**: 화면 우하단 액션 버튼이 빛남 + NPC 위 `!` 바운스 강화
- **애니메이션**: 프롬프트 등장 시 아래→위 바운스 (200ms, easeOut)

## 5. 대화 박스 상세 설계

### 5.1 레이아웃

```
┌──────────────────────────────────────────────────┐
│  ╔═══════════╗                                   │
│  ║           ║  전도자 (Evangelist)               │ ← 화자 이름 (accent 색)
│  ║ Portrait  ║                                   │
│  ║  32×32    ║  "이 길을 따라가라.                │ ← 타이핑 텍스트
│  ║           ║   빛이 보이면 그곳으로 달려가라."  │
│  ╚═══════════╝                            [▼]    │ ← 진행 표시 (깜빡임)
│                                                  │
│  ┌──────────────┐  ┌──────────────┐              │ ← 선택지 (있을 때만)
│  │ 1. 빛을 향해 │  │ 2. 두렵지만  │              │
│  │    달린다    │  │    한 발만   │              │
│  └──────────────┘  └──────────────┘              │
└──────────────────────────────────────────────────┘
```

### 5.2 포트레이트 (Portrait)

- **크기**: 32x32 원본, 대화 박스 내 2x 확대 (64x64 물리 픽셀, nearest-neighbor)
- **프레임**: 양피지 스타일 테두리 (2px 금색 테두리)
- **감정 전환**: Ink 태그 `# EMOTION: angry`에 따라 포트레이트 텍스처 스왑
  - 전환 애니메이션: 0.15초 crossfade
- **없는 경우**: 포트레이트 영역 축소, 텍스트가 전체 너비 사용

### 5.3 타이핑 이펙트

| 모드 | 속도 | 트리거 |
|------|------|--------|
| Normal | 40ms/글자 | 기본 |
| Slow | 80ms/글자 | `# TYPING: slow` (슬픔, 심사숙고) |
| Fast | 20ms/글자 | `# TYPING: fast` (긴장, 흥분) |
| Letter-by-letter | 200ms/글자 | `# TYPING: dramatic` (강조, 충격) |
| Instant | 0ms | 클릭으로 스킵 |

- **타이핑 효과음**: 글자마다 짧은 SFX (톤은 화자별로 다름)
  - 크리스천: 부드러운 클릭
  - 전도자: 낮은 목소리 톤
  - 아볼론: 거친 노이즈
  - 빛나는 자들: 맑은 벨 소리

### 5.4 선택지 UX

| 플랫폼 | 선택 방식 |
|--------|----------|
| 키보드 | 방향키 위/아래 + Enter. 현재 선택에 금색 테두리 |
| 마우스 | 호버 시 금색 하이라이트, 클릭 선택 |
| 터치 | 탭 선택, 선택지 높이 최소 32px (내부 해상도 기준) |
| 게임패드 | D-Pad 위/아래 + A 버튼 |

#### 선택지 등장 애니메이션

- 일반 선택: 아래에서 올라옴, 0.2초 간격으로 순차 등장
- 중요 선택 (`# CHOICE_WEIGHT: heavy`): 화면 디밍 → 0.5초 정지 → 선택지 하나씩 1초 간격 등장 → 배경 저음 드론
- 숨겨진 선택 (스탯 게이팅): 금색 빛 이펙트와 함께 등장, "★" 마크

#### 선택지 잠금 표시

- 스탯 미달 시: 선택지 회색 처리 + 자물쇠 아이콘 + "(지혜 30 필요)" 툴팁
- 데스크탑: 호버 시 툴팁
- 모바일: 탭 시 3초간 요구 조건 토스트

## 6. 터치 UX 패턴

### 6.1 가상 조이스틱

#### 사양

| 속성 | 기본값 | 커스터마이징 범위 |
|------|--------|-----------------|
| 크기 (외곽 원) | 64px 물리 | 48–80px |
| 크기 (내부 원) | 24px 물리 | 16–32px |
| 투명도 | 0.6 | 0.2–1.0 |
| 위치 | 좌하단 | 좌하단/좌중단 (고정) |
| 데드존 | 8px | 4–12px |
| 모드 | Dynamic (터치 시 위치 결정) | Dynamic / Fixed |

#### 동작

- **Dynamic 모드**: 터치 시작 위치에 조이스틱 생성, 손가락 떼면 0.3초 후 사라짐

- **입력 변환**: 조이스틱 방향 → 4방향 또는 8방향 (설정 가능, 기본 4방향)

## 9. 접근성

### 9.1 색약 모드

| 모드 | 변경 사항 |
|------|----------|
| OFF | 기본 컬러 |
| 적록색약 (Deuteranopia) | 녹색 계열 → 청색 계열, 적색 → 주황 |
| 청황색약 (Tritanopia) | 파란색 → 보라, 노랑 → 분홍 |
| 전색맹 (Achromatopsia) | 모든 스탯 바에 패턴(빗금/점/격자) 추가 |

스탯 바에는 색상 외에 **아이콘**과 **약어**를 항상 표시하여 색상만으로 구분할 필요가 없도록 한다.

### 9.2 폰트 크기 조절

| 설정 | 배율 | 대화 `--text-base` |
|------|------|-------------------|
| 작음 | 0.85x | 7px |
| 보통 | 1.0x | 8px |
| 큼 | 1.25x | 10px |

큰 폰트 사용 시 대화 박스 높이 자동 확장.

### 9.3 키보드 내비게이션

모든 인터랙티브 요소에 포커스 인디케이터 (금색 점선 테두리).

| 키 | 기능 |
|----|------|
| Tab / Shift+Tab | 포커스 이동 |
| Enter / Space | 선택 / 확인 |
| Escape | 뒤로 / 닫기 |
| 방향키 | 메뉴 내 이동 / 게임 내 이동 |

### 9.4 스크린 리더 힌트

- 모든 UI 요소에 `aria-label` 속성 (DOM 오버레이 방식의 경우)
- 스탯 변화 시 `aria-live="polite"` 영역에 텍스트 업데이트
- 대화 텍스트는 타이핑 완료 후 `aria-live` 영역에 전체 텍스트 게시

### 9.5 모션 감소

- `prefers-reduced-motion: reduce` 미디어 쿼리 감지
- 해당 시: 화면 흔들림 OFF, 파티클 감소, 전환 효과 단순화 (fade only)

## 10. 구현 참고사항

### 10.1 UI 레이어 구조

```
Layer 0: 게임 월드 (Phaser Scene)
Layer 1: 월드 UI (NPC 프롬프트, 이름표 — Scene 내 Phaser 오브젝트)
Layer 2: HUD (스탯 바, 미니맵 — 고정 위치 Phaser 오브젝트, scrollFactor 0)
Layer 3: 대화 박스 (Phaser UI Scene or DOM overlay)
Layer 4: 토스트/알림 (Phaser or DOM)
Layer 5: 메뉴/설정 (별도 Phaser Scene overlay or DOM)
Layer 6: 장면 전환 (fullscreen rectangle, highest depth)
```

### 10.2 DOM vs Phaser UI 전략

| 요소 | 렌더 방식 | 이유 |
|------|----------|------|
| HUD 스탯 바 | Phaser | 게임과 동일 픽셀 그리드, 애니메이션 제어 |
| 대화 박스 | Phaser | 포트레이트, 타이핑 이펙트, 게임 내 연출 통합 |
| 선택지 | Phaser | 게임 세계와 일관된 비주얼 |
| 토스트 | Phaser | 게임 이펙트와 동기화 |
| 메인 메뉴 | Phaser Scene | 배경 애니메이션, 전환 효과 |
| 설정 패널 | DOM overlay | 슬라이더/드롭다운 등 네이티브 컨트롤 활용 |
| 월드맵 | Phaser Scene | 맵 렌더링, 애니메이션 |

### 10.3 터치 입력 처리

```typescript
interface TouchConfig {
  joystickSize: number;      // 물리 px
  joystickDeadzone: number;  // 물리 px
  joystickMode: 'dynamic' | 'fixed';
  joystickOpacity: number;   // 0–1
  buttonSize: number;        // 물리 px
  buttonOpacity: number;     // 0–1
  gestureThreshold: number;  // 스와이프 인식 최소 거리 (물리 px)
}
```

### 10.4 반응형 감지

```typescript
interface DeviceInfo {
  isTouchDevice: boolean;
  isPortrait: boolean;
  screenWidth: number;
  screenHeight: number;
  safeArea: { top: number; bottom: number; left: number; right: number };
  pixelRatio: number;
  uiMode: 'desktop' | 'tablet' | 'mobile-landscape' | 'mobile-portrait';
}
```

`resize` 이벤트와 `orientationchange` 이벤트에서 `DeviceInfo` 업데이트 → EventBus로 `UI_MODE_CHANGED` 이벤트 발행 → 각 UI 컴포넌트가 레이아웃 재배치.
